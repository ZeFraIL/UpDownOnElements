## 🎯 Цель проекта
• **Какую задачу решает приложение?**  
Приложение позволяет пользователю увеличивать или уменьшать значение счетчика путем физического наклона устройства.

• **Почему эта задача важна?**  
Демонстрирует интеграцию аппаратных датчиков (акселерометра) с пользовательским интерфейсом и обработку входных данных от физических движений в реальном времени.

• **Целевая аудитория**  
Студенты и разработчики, изучающие работу с сенсорами в Android.
________________________________________
## 📌 Требования к приложению
**Функциональные требования**  
• Обнаружение наклона устройства по оси X.  
• Увеличение счетчика при наклоне в одну сторону (положительная X).  
• Уменьшение счетчика при наклоне в противоположную сторону (отрицательная X).  
• Обновление дисплея в режиме реального времени.

**Нефункциональные требования**  
• **Производительность:** Быстрая реакция на события датчика.  
• **Удобство использования:** Простой, интуитивно понятный интерфейс с крупным, читаемым текстом.  
• **Надёжность:** Корректная регистрация и отмена регистрации датчика во время жизненного цикла Activity.

## 🧠 Общая архитектура
• **Выбранный подход:**  
– Паттерн MVC (Model-View-Controller).

• **Почему выбран именно он:**  
Для простого одноэкранного приложения этот подход позволяет сохранить логику прямолинейной и понятной без излишней сложности.

• **Основные компоненты системы:**  
– `MainActivity`: Выступает и как контроллер (обработка событий сенсора), и как менеджер представления (обновление UI).

## 🧩 UML-диаграмма
[MainActivity] –> [SensorManager]  
[MainActivity] –> [TextView (UI)]  
*MainActivity реализует SensorEventListener для получения обновлений от акселерометра.*
________________________________________
## 🧩 Подробное описание классов
### 📌 Класс: MainActivity
**Роль:**  
Основная точка входа и обработчик логики приложения.  
**Ответственность:**  
Инициализация UI, управление жизненным циклом сенсора и обработка данных акселерометра.  
**Основные методы:**  
- `onCreate()` — Инициализирует представление и менеджер сенсоров.
- `onSensorChanged()` — Обрабатывает значения акселерометра и обновляет счетчик.
- `onDestroy()` — Отменяет регистрацию слушателя сенсора для экономии ресурсов.

**Взаимодействие с другими классами:**  
Взаимодействует с `SensorManager` и `Sensor` для получения данных от оборудования.
________________________________________
## 🔄 Диаграмма работы приложения
1. Пользователь открывает приложение.
2. `MainActivity` начинает прослушивание акселерометра.
3. Пользователь наклоняет телефон.
4. `onSensorChanged` определяет величину наклона.
5. Если величина превышает порог, счетчик увеличивается или уменьшается.
6. `TextView` обновляется с новым значением.

## 🎨 UI/UX анализ
• **Почему интерфейс сделан именно так:**  
Чистый и центрированный дизайн, чтобы сосредоточить внимание пользователя на значении счетчика.

• **Какие принципы использованы:**  
– **Простота:** Никаких лишних кнопок или меню.  
– **Логичность:** Движение напрямую преобразуется в числовые изменения.  
– **Доступность:** Крупный шрифт для счетчика.

• **Что можно улучшить:**  
Добавить визуальные индикаторы (стрелки), показывающие, в каком направлении нужно наклонять устройство.
________________________________________
## ⚙️ Работа с потоками
• **Используются:**  
– Главный (UI) поток.  
• **Почему выбран этот способ:**  
События датчиков легкие и по умолчанию доставляются системой в главный поток.  
• **Как предотвращаются:**  
– **Зависания (ANR):** Минимальная обработка внутри слушателя.  
– **Утечки памяти:** Отмена регистрации слушателя в `onDestroy()`.
________________________________________
## 💾 Работа с данными
• **Где хранятся данные:**  
Переменная в памяти (`int number`).  
• **Почему выбран этот способ:**  
Постоянное хранение не было основным требованием для этой демонстрации.  
• **Как обеспечивается корректность:**  
Использование флагов (`isCountingUp`, `isCountingDown`) для предотвращения многократных быстрых приращений за одно событие наклона.
