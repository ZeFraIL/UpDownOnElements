# Описание класса: MainActivity

________________________________________
## 1. Общая информация
*   **Имя класса:** `MainActivity`
*   **Тип:** Activity (Активность — компонент, который отвечает за окно приложения, с которым взаимодействует пользователь).
*   **Назначение:** Этот класс является «сердцем» приложения. Он управляет главным экраном и отслеживает физический наклон телефона с помощью датчика под названием **Акселерометр**. В зависимости от того, в какую сторону наклонен телефон, класс увеличивает или уменьшает значение счетчика.
*   **Взаимодействие:** Он взаимодействует с системной службой Android `SensorManager` для получения данных от «железа» и обновляет разметку `activity_main.xml`, чтобы показать результат пользователю.

________________________________________
## 2. Переменные (Поля класса)

| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `tvNumber` | `TextView` | Текстовое поле для отображения текущего числа на экране. | `onCreate`, `onSensorChanged` |
| `sensorManager` | `SensorManager` | Системный менеджер, дающий доступ ко всем датчикам устройства. | `onCreate`, `onDestroy` |
| `accelerometer` | `Sensor` | Объект, представляющий конкретный датчик (акселерометр). | `onCreate` |
| `number` | `int` | Хранит целое число (значение нашего счетчика). | `onSensorChanged` |
| `SHAKE_THRESHOLD_POSITIVE` | `float` | Порог срабатывания для наклона вперед (константа: 2.5). | `onSensorChanged` |
| `SHAKE_THRESHOLD_NEGATIVE` | `float` | Порог срабатывания для наклона назад (константа: -2.5). | `onSensorChanged` |
| `isCountingUp` | `boolean` | Флаг-предохранитель, чтобы число не росло слишком быстро при одном наклоне. | `onSensorChanged` |
| `isCountingDown` | `boolean` | Флаг-предохранитель, чтобы число не уменьшалось слишком быстро. | `onSensorChanged` |

________________________________________
## 3. Методы класса

### Название метода: `onCreate`
*   **Тип метода:** `protected`
*   **Возвращаемое значение:** `void` (ничего не возвращает)
*   **Параметры:**
    *   `savedInstanceState` (`Bundle`): Содержит сохраненные данные, если активность создается заново.
*   **Что делает метод:**
    1. Устанавливает визуальный интерфейс (`activity_main.xml`).
    2. Находит текстовый элемент (`tvNumber`) на экране.
    3. Запрашивает у системы доступ к датчикам.
    4. Пытается найти акселерометр. Если его нет в телефоне, выводит предупреждение (Toast).
    5. Регистрирует «слушателя», чтобы приложение начало получать данные о движении.
*   **Когда вызывается:** Автоматически системой при запуске приложения.
*   **Что важно понимать:** Это этап настройки. Если здесь не инициализировать датчик, приложение не будет реагировать на наклоны.

### Название метода: `onDestroy`
*   **Тип метода:** `protected`
*   **Возвращаемое значение:** `void`
*   **Что делает метод:** Приказывает системе перестать отправлять данные от датчика приложению.
*   **Когда вызывается:** При закрытии приложения пользователем или системой.
*   **Что важно понимать:** **Критически важно для экономии заряда батареи.** Если не «отписаться» от датчика, телефон будет продолжать его опрашивать даже после закрытия приложения, быстро разряжая аккумулятор.

### Название метода: `onSensorChanged`
*   **Тип метода:** `public`
*   **Возвращаемое значение:** `void`
*   **Параметры:**
    *   `event` (`SensorEvent`): Объект с данными от датчика (координаты наклона).
*   **Что делает метод:**
    1. Проверяет, что данные пришли именно от акселерометра.
    2. Считывает значение по оси `x` (наклон вправо-влево).
    3. Если `x > 2.5` и мы еще не засчитали этот наклон, прибавляет 1 к числу.
    4. Если `x < -2.5` и наклон не засчитан, отнимает 1.
    5. Когда телефон возвращается в ровное положение, сбрасывает флаги, чтобы можно было наклонить снова.
*   **Когда вызывается:** Каждый раз, когда телефон шевелится (много раз в секунду).
*   **Что важно понимать:** Этот метод должен работать очень быстро. Если здесь написать тяжелый код, приложение будет «тормозить».

________________________________________
## 4. Жизненный цикл
*   **`onCreate()`**: Вызывается первым. Здесь мы готовим всё (связываем переменные с UI, запускаем датчики).
*   **`onDestroy()`**: Вызывается последним. Здесь мы «убираем за собой» (выключаем датчики).

________________________________________
## 5. Взаимодействие с интерфейсом (UI)
*   **Элементы:** `TextView` (id: `tvNumber`).
*   **Связь:** Через команду `findViewById(R.id.tvNumber)`.
*   **События:** Обрабатывает не нажатия на кнопки, а **события датчика** через интерфейс `SensorEventListener`.

________________________________________
## 6. Взаимодействие с другими компонентами
*   **Оборудование:** Напрямую общается с акселерометром через `SensorManager`.
*   **Toast:** Использует класс `Toast` для вывода всплывающих уведомлений, если датчик не найден.

________________________________________
## 7. Общая логика класса
Класс работает как мостик между физическим миром и цифровым экраном.
1. Приложение запускается и начинает «слушать» гравитацию.
2. Когда вы наклоняете телефон, «железо» отправляет координаты.
3. Класс проверяет: «Достаточно ли сильный этот наклон?».
4. Если да — меняет число и приказывает экрану показать новое значение.

________________________________________
## 8. Упрощенное объяснение
**«Объяснение простыми словами»**
Представьте, что телефон — это **физические весы** или **качели-балансир**.
*   Представьте, что в центре экрана лежит шарик.
*   Когда вы наклоняете телефон вправо, шарик катится и нажимает на кнопку «Плюс».
*   Когда наклоняете влево — на кнопку «Минус».
*   Флаги `isCountingUp/Down` работают как «задержка» — они нужны, чтобы одно движение считалось как одно нажатие, иначе цифры бы крутились с бешеной скоростью, как в игровом автомате.

**Примечание по качеству кода:**
*   **Проблема:** Пороги `2.5f` жестко прописаны в коде. Было бы лучше вынести их в настройки.
*   **Улучшение:** В современном Android лучше регистрировать датчики в `onStart`, а отключать в `onStop`, чтобы не тратить ресурсы, когда приложение свернуто, но не закрыто.
