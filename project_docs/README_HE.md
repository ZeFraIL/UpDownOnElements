## 🎯 מטרת הפרויקט
• **איזו משימה האפליקציה פותרת?**  
האפליקציה מאפשרת למשתמש להעלות או להוריד את ערך המונה על ידי הטיה פיזית של המכשיר.

• **מדוע משימה זו חשובה?**  
היא מדגימה כיצד לשלב חיישני חומרה (מד תאוצה - Accelerometer) עם ממשק המשתמש ולטפל בקלט נתונים בזמן אמת מתנועה פיזית.

• **קהל יעד**  
סטודנטים ומפתחים הלומדים ניהול חיישנים באנדרואיד.
________________________________________
## 📌 דרישות האפליקציה
**דרישות פונקציונליות**  
• זיהוי הטיית המכשיר לאורך ציר ה-X.  
• הגדלת המונה בעת הטיה לכיוון אחד (X חיובי).  
• הקטנת המונה בעת הטיה לכיוון הנגדי (X שלילי).  
• עדכון התצוגה בזמן אמת.

**דרישות לא פונקציונליות**  
• **ביצועים:** תגובה מהירה לאירועי חיישן.  
• **חווית משתמש:** ממשק פשוט ואינטואיטיבי עם טקסט גדול וקריא.  
• **אמינות:** רישום וביטול רישום נכון של החיישן במהלך מחזור החיים של ה-Activity.

## 🧠 ארכיטקטורה כללית
• **גישה נבחרת:**  
– תבנית MVC (Model-View-Controller).

• **מדוע נבחרה דווקא היא:**  
עבור אפליקציה פשוטה בעלת מסך אחד, גישה זו שומרת על הלוגיקה ישירה וקלה להבנה ללא מורכבות מיותרת.

• **מרכיבי המערכת העיקריים:**  
– `MainActivity`: משמשת הן כבקר (טיפול באירועי חיישן) והן כמנהלת התצוגה (עדכון ה-UI).

## 🧩 תיאור מחלקות מפורט
### 📌 מחלקה: MainActivity
**תפקיד:**  
נקודת הכניסה הראשית ומנהלת הלוגיקה של האפליקציה.  
**אחריות:**  
אתחול ממשק המשתמש, ניהול מחזור החיים של החיישן ועיבוד נתוני מד התאוצה.  
**שיטות עיקריות:**  
- `onCreate()` — מאתחלת את התצוגה ואת מנהל החיישנים.
- `onSensorChanged()` — מעבדת את ערכי מד התאוצה ומעדכנת את המונה.
- `onDestroy()` — מבטלת את רישום המאזין לחיישן כדי לחסוך במשאבים.

**אינטראקציה עם מחלקות אחרות:**  
מתקשרת עם `SensorManager` ו-`Sensor` כדי לקבל נתוני חומרה.

## 🔄 תזרים עבודה של האפליקציה
1. המשתמש פותח את האפליקציה.
2. `MainActivity` מתחילה להאזין למד התאוצה.
3. המשתמש מטה את הטלפון.
4. `onSensorChanged` מזהה את עוצמת ההטיה.
5. אם העוצמה עולה על הסף, המונה גדל או קטן.
6. ה-`TextView` מתעדכן בערך החדש.
